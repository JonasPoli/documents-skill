## Nova Abordagem

Em vez de gerar mockups visuais, todas as telas e seções de um orçamento devem ser representadas por **descrições textuais detalhadas** (wireframes textuais) inseridas diretamente no documento Markdown.

Para cada tela/seção:
1. Descreva todos os componentes visuais (cards, tabelas, formulários, botões, ícones, etc.).
2. Detalhe a disposição e hierarquia dos elementos.
3. Explique a funcionalidade de cada componente interativo.
4. Descreva a experiência do usuário ao navegar pela tela.
