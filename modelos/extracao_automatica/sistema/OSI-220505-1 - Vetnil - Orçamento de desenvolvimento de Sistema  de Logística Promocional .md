Descrição do orçamento
O objetivo deste documento é descrever o orçamento para o desenvolvimento do sistema de gerenciamento da
logística promocional da Vetnil.

Cenário Geral
De forma geral, o sistema deverá permitir que seja gerenciado os protocolos de solicitação de matérias pelos
gerentes de marketing e pelos membros de sua equipe.

Acesso ao sistema
O acesso ao sistema será restrito aos usuários previamente cadastrados pelo administrador da Vetnil. Para
acessar as áreas do sistema, os usuários deverão ser identificados por um painel de identificação, com login e
senha.

Painel Administrativo
Existirá, no novo sistema, um primeiro usuário que definirá quem são as outras entidades que poderão acessar
o sistema. Para isso, o novo sistema possuirá um painel administrativo por onde será possível gerenciar as
seguintes áreas:

Dashboard do Administrador
Neste painel serão apresentados:
●​
●​
●​
●​

A quantidade total de pessoas cadastradas;
A quantidade total de gerentes cadastrados;
A listagem de saldo restante por cada gerente;
A próxima janela de solicitação de brindes;

Também será apresentado um gráfico de linha dos últimos 6 meses com a evolução do total, dos valores
envolvidos nos pedidos.
Ainda no dashboard será apresentada a listagem de usuários cadastrados que aguardam a moderação pelo
usuário operador.

Moderação do Cadastro de Usuários
Quando o administrador do sistema acessar o painel de moderação de cadastro de usuários, o sistema exibirá
uma lista de usuários cadastrados que estão aguardando a aprovação.
Quando o administrador escolher moderar um usuário, o sistema apresentará um formulário com todos os
campos preenchidos pelo usuário no ato do cadastro. O administrador deverá analisar visualmente os dados e
tomar uma das decisões:
●​ Aprovar o cadastro do usuário
○​ Ao realizar esta ação, o administrador deverá o tipo de usuários e as áreas que possuirá acesso
ao sistema
○​ O usuário aprovado receberá um e-mail informado que seu acesso está ativo.
●​ Reprovar o cadastro do usuário
○​ Para reprovar, o administrador deverá preencher um formulário com os motivos da reprova.
○​ O usuário reprovado receberá um e-mail dizendo que seu cadastro foi reprovado, com os
motivos informados pelo administrador. O acesso ao sistema por este usuário estará
bloqueado.

Cadastro de Usuários
O administrador poderá visualizar a lista dos usuários cadastrados no sistema e editá-los. Poderá alterar os
dados cadastrados pelo usuário, alterar a lista permissões associadas e aprovar e reprovar o cadastro.

Cadastro de Gerentes
Os gerentes serão um tipo especial de usuários do sistema que possuirão suas verbas cadastradas. Um gerente
não pode cadastrar sua própria verba. Esta funcionalidade será atribuída aos operadores responsáveis.
Além dos campos descritos no cadastro geral de usuários, um gerente possuirá um gerenciamento de suas
verbas.
Cada verba informada sera definida como:
●​ tipo
○​ Verba receituário do futuro - Verba entra trimestralmente acumulativa até o final do ano
○​ Verba material promocional - Verba entra trimestralmente - Não acumula
○​ Verba produto acabado - Verba entra trimestralmente acumulativa até o final do ano
●​ descrição
●​ data de início
●​ data de término
●​ acumula
○​ Não acumula, estingue no fim do mês.
○​ Até o fim ano

Todos os itens possuirão:
●​ Nome
●​ Código de identificação
●​ Valor unitário (em reais)

Atualização automática dos itens
Existirá um procedimento automático que importará os itens do sistema da VETNIL. Este procedimento deverá
identificar se o item a ser importado já existe no sistema. Se não existir, adiciona, se existir, atualiza.
O procedimento apenas atualizará itens que são mantidos por atualização, não alterando os itens adicionados
manualmente.

Cadastro de Janela de compra
A solicitação de brindes está associada a uma janela de compras. As janelas de compras deverão ser
gerenciados por um operador autorizado.
Cada janela de compra será mantida pelos campos:
●​ Nome da janela
●​ Data de início da janela
●​ Data do fim da janela
●​ Quantidade de dias para avisar do início da janela
●​ Quantidade de dias para avisar do fim da janela
O cadastro das janelas de compras deverá impedir que janelas sejam sobrepostas, ou seja:
●​ Uma janela de compras só poderá começar após encerrada a janela anterior
●​ Uma janela de compras devera finalizar antes do início da janela posterior

E-mail transacional de status
Quando um pedido for criado (Receituário do Futuro, Pedido de Material ou Pedido de brindes) o sistema
enviará um e-mail com todas as informações do solicitante, do pedido e dos itens solicitados e enviará ao
solicitante.
Cada alteração de status gerará um novo e-mail transacional sobre o pedido.

Protocolos
O usuário do tipo operador autorizado a gerenciar os protocolos possuirá acesso a este painel.
Neste painel serão exibidos todos os pedidos que estão sendo processados.
Cada protocolo será apresentado com:
●​ Número do protocolo
●​ Data e hora da solicitação
●​ Tipo de solicitação (tipo de pedido)
●​ Usuário solicitante
●​ Itens na solicitação
●​ Valor total da solicitação
Ao editar um protocolo, o operador poderá editar seu campo de "observações internas" que não será visível ao
usuário que solicitou.
O operador poderá alterar o status do protocolo para:
Sobre os Status dos pedidos:
●​ Pedido registrado
○​ Preencher um campo texto para operador
○​ Preencher um campo que será enviado ao solicitante
●​ Pedido sendo processado
○​ Separação
■​ Preencher um campo texto para operador
■​ Preencher um campo que será enviado ao solicitante
○​ Nota Fiscal emitida
■​ Subir o PDF da nota
■​ Preencher um campo texto para operador
■​ Preencher um campo que será enviado ao solicitante
●​ Pedido em trânsito (transportadora)
○​ Preencher um campo texto para operador
○​ Preencher um campo que será enviado ao solicitante

○​ Preencher um campo que será enviado ao solicitante

Serviços necessários:
●​

Gerenciamento em estrutura desconectada
○​ Este serviço implica na existência de um serviço de gerenciamento em outra estrutura,
separada de onde o sistema ficará hospedado. Isso permite que o gerenciamento do servidor
seja realizado mesmo que o servidor esteja fora do ar.

●​

Ambiente de homologação
○​ Será disponibilizado um ambiente de testes, onde poderão ser realizados testes no sistema. O
ambiente de homologação será disponibilizado em um endereço paralelo para que testes de
implementação possam ser realizados antes de irem ao ar definitivamente no ambiente de
produção, garantindo maior segurança ao processo.

●​

Serviço de backup diário
○​ Este serviço garante que o sistema e seu banco de dados sejam copiados uma vez por dia para
outro servidor separado da hospedagem principal. Isso garante que, em caso de alguma
catástrofe, o backup do sistema seja acessado mesmo que a estrutura do sistema não esteja
mais disponível.
○​ Poderão ser disponibilizados os 15 últimos backups mediante solicitação devidamente
formalizada.
○​ Também será possível restaurar um backup no sistema de homologação.

●​

Análise externa do servidor com Monitoramento 24h
○​ Será implantado um sistema de monitoramento, instalado em diversos computadores
espalhados pelo mundo que nos informarão imediatamente caso o serviço fique indisponível.
Isso permite que as ações de recuperação possam ser executadas o mais rápido possível.

●​

Hospedagem de arquivos na Amazon S3
○​ Todos os arquivos enviados ao sistema serão armazenados no S3 da Amazon, construído para
armazenar e recuperar qualquer volume de dados de qualquer local. Isso permitirá que o
volume de dados do sistema e seu banco de dados sejam bem menores, o que possibilitará
uma rápida ação de backup e de recuperação deste backup.
○​ O gerenciamento do espaço disponível do S3 é elástico, pago por uso.

●​

Restauração de cópia do sistema
○​ Caso seja necessário, uma cópia do servidor poderá ser instanciada com os dados de backup.
