DESENVOLVIMENTO DE SISTEMA
INTRODUÇÃO
A WAB Agência Digital apresenta esta proposta para o desenvolvimento de um sistema de pedidos B2B para a
empresa W.U.​
O sistema permitirá a realização de pedidos por meio de três áreas distintas:
●​ Área Pública – onde visitantes poderão visualizar a listagem de produtos, realizar buscas e filtrar por
categorias;
●​ Área do Cliente Logado – com acesso a detalhes dos produtos, funcionalidade de carrinho e finalização
de pedidos;
●​ Área do Administrador – para gestão dos pedidos, visualização de dashboards com indicadores,
controle de sincronismo, gerenciamento de banners, páginas institucionais e exportação de dados.
O objetivo deste documento é descrever minuciosamente as funcionalidades, os módulos do sistema,
integrações e requisitos técnicos, servindo como base para o orçamento e posterior desenvolvimento do
projeto.

CENÁRIO GERAL
O sistema de pedidos B2B possibilitará que clientes realizem pedidos de produtos via uma interface intuitiva,
enquanto administradores terão uma área robusta para monitorar, gerenciar e sincronizar os dados dos
pedidos e produtos. A integração com o sistema local do cliente será realizada via APIs seguras, permitindo a
injeção, atualização, desativação e inserção de produtos, bem como a sincronização de categorias e preços por
cliente.
Aspectos de segurança, performance e responsividade serão priorizados, garantindo comunicação
criptografada (SSL) e acesso por tokens.

ACESSO AO SISTEMA
Para garantir a segurança, será utilizada a tecnologia de verificação em duas etapas. O usuário que efetuar
login receberá imediatamente um e-mail contendo um código de segurança. Esse processo se repetirá a cada
novo login.

RESPONSIVIDADE
O sistema será desenvolvido seguindo os padrões de visualização responsiva, o que significa que o layout será
otimizado para adaptar-se a diferentes resoluções de tela. Isso incluirá telas grandes de até 1680px de largura,
iPads em modo paisagem e retrato, iPhones em modo paisagem e retrato, entre outros dispositivos.

SISTEMA DE SINCRONISMO E
INTEGRAÇÃO (APIs)
O sistema integrará dados do sistema local da W.U. via APIs REST seguras, permitindo:

●​ Injeção de Produtos:
○​ Rota para envio de todos os produtos (único POST) que, ao ser processado, atualiza a base
on-line.
○​ Análise via código para identificar alterações, inclusões e desativa produtos não enviados.

●​ Atualização, Desativação e Inserção:
○​ Rotas específicas para atualização e desativação de produtos conforme alteração ou remoção
no sistema local.
○​ Rota para inserção de novos produtos.

●​ Sincronismo Complementar:
○​ Sincronização de categorias e atualização dos preços por cliente.
○​ Histórico de sincronismos com consulta por data/hora e listagem completa dos pedidos.

●​ Logs e Auditoria:
○​ Registro de todas as comunicações recebidas (injeção, atualização, desativação e inserção) com
armazenamento em arquivos de texto organizados por ano/mês/dia.

●​ Segurança:
○​ Proteção das rotas via autenticação por token e validação robusta dos dados recebidos.
○​ Controle de versão dos dados para facilitar a reversão em caso de inconsistências.

Sistema Local
Para garantir uma integração consistente e sem falhas entre o sistema local e o sistema online, o sistema local
precisará ser ajustado para enviar os dados de forma estruturada e na ordem correta. O fluxo de integração
deverá ocorrer da seguinte forma:
1.​ Sincronização dos Clientes:
○​ Objetivo: Garantir que todos os clientes cadastrados no sistema local estejam refletidos no
sistema online.
○​ Processo:
■​ Injeção em Massa: Inicialmente, o sistema local deverá enviar a listagem completa de
clientes através da rota /api/clients/inject. Essa operação verificará quais
clientes já existem, atualizará os registros existentes, inserirá novos clientes e
removerá ou desativará aqueles que não constarem na listagem atual.
■​ Atualizações Individuais: A cada alteração, cadastro ou remoção de um cliente, o
sistema local deverá acionar os endpoints específicos:
■​ /api/clients/insert para novos cadastros,
■​ /api/clients/update para alterações nos dados existentes,
■​ /api/clients/remove para remoção ou inativação.
○​ Quando Acessar: Sempre que ocorrer um cadastro novo ou atualização dos dados dos clientes,
além da sincronização periódica em lote.
2.​ Sincronização de Categorias:
○​ Objetivo: Atualizar e organizar as categorias dos produtos no sistema online para que reflitam
fielmente a estrutura do sistema local.
○​ Processo:
■​ Através do endpoint /api/categories/sync, o sistema local enviará a listagem
de categorias atualizada.
○​ Quando Acessar: Periodicamente ou sempre que houver modificações na organização das
categorias dos produtos.
3.​ Sincronização dos Produtos:
○​ Objetivo: Atualizar a base de produtos do sistema online com as informações do sistema local,
garantindo que qualquer modificação seja refletida.
○​ Processo:
■​ Injeção em Massa: O sistema local deverá enviar a listagem completa de produtos por
meio do endpoint /api/products/inject. Este processo identificará, com base
em um identificador único, se um produto foi alterado, se é novo ou se deve ser
desativado (quando não consta na listagem atual).
■​ Atualizações Individuais:
■​ Para atualizar um produto existente, utilizar o endpoint
/api/products/update.
■​ Para inserir um novo produto, utilizar o endpoint
/api/products/insert.
■​ Caso o produto seja removido do sistema local, acionar o endpoint
/api/products/deactivate.
○​ Quando Acessar: Sempre que houver atualizações ou alterações no catálogo de produtos,
além de sincronizações periódicas.

4.​ Sincronização dos Preços dos Produtos para os Clientes:
○​ Objetivo: Garantir que os preços personalizados, de acordo com as condições comerciais e
descontos específicos para cada cliente, estejam atualizados no sistema online.
○​ Processo:
■​ O sistema local deverá acessar o endpoint /api/products/price-sync,
enviando os dados de preços personalizados para cada cliente.
○​ Quando Acessar: Sempre que houver alterações nos preços dos produtos ou nos acordos
comerciais específicos para um cliente, após a confirmação da sincronização dos clientes e
produtos.

Resumo do Fluxo de Integração:
1.​ Clientes: Injeção e atualização dos dados dos clientes (rotas /api/clients/inject,
/api/clients/update, /api/clients/insert e /api/clients/remove), garantindo
que o sistema online contenha todos os clientes ativos do sistema local.
2.​ Categorias: Sincronização da organização das categorias através do endpoint
/api/categories/sync.
3.​ Produtos: Sincronização completa dos produtos (rotas /api/products/inject,
/api/products/update, /api/products/insert e /api/products/deactivate), de
modo a refletir a base de produtos atualizada e desativar aqueles que não estão mais presentes.
4.​ Preços Personalizados: Atualização dos preços dos produtos para cada cliente, utilizando a rota
/api/products/price-sync, para que os valores específicos sejam aplicados corretamente.
Essas alterações e integrações devem ser acionadas de forma automática pelo sistema local sempre que
ocorrerem modificações relevantes ou periodicamente, garantindo a integridade e a atualização contínua dos
dados entre o sistema offline e o online.
