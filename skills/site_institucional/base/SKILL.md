# Skill: Gerador de Orçamento Site Institucional (Genérico)

**REGRA DE OURO:** A "Regra de Ouro" da WAB: Se o orçamento não tiver um print real de como o sistema/site vai ficar, ele é INVÁLIDO.
**NOVO:** Você DEVE selecionar componentes da [`galeria.md`](file:///Volumes/Dados/work/documentos/biblioteca_de_componentes/galeria.md) para montar os mockups.
Exemplo: Se o cliente quer um site de cursos, use o `header_1`, `features_5` e `footer_10`.
um mockup visual para **cada seção da landing page** (Hero, Quem Somos, Serviços, etc).

## 1. DESCOBERTA
- "Qual a ideia central do site?"
- "Nome do Cliente e Tom desejado?"

## 2. A HIERARQUIA DA EXPANSÃO
Siga estritamente a **ESPINHA DORSAL FUNDECITRUS** listada no `MASTER_TEMPLATE.md`.

As seções obrigatórias que você inventará o preenchimento, usando linguagem técnica e comercial:
1. Cenário Geral (Dores do cliente que você deve inferir a partir do input de 1 linha dele).
2. Tecnologia Sugerida (Liste vantagens de design responsivo, mobile first e acessibilidade W3C).
3. Estrutura do Sistema/Site (Área Pública - Home, Institucional, Relatórios).
4. Prazo, Condições Comerciais, Itens não Inclusos.

## 3. GERAÇÃO E SALVAMENTO
Seja eloquente. Crie wireframes textuais completos de cabeçalho a rodapé para pelo menos 5 páginas padrão do site inventadas por você baseadas no setor do cliente.
Salve em `/Volumes/Dados/work/documentos/orcamentos_gerados/` como `SITE_COMPLETO_[CLIENTE].md`.
