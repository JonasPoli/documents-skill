# Skill Completa: Orçamento de Sistema Symfony

Esta skill transforma necessidades do cliente em propostas técnicas robustas, detalhadas e profissionais para o desenvolvimento de sistemas personalizados utilizando o framework Symfony.

> [!IMPORTANT]
> **REGRA DE OURO:** Nunca resuma as informações. Utilize a "Anti-Summary Rule". Detalhe cada funcionalidade, tela e processo. Proibições: "etc", "...", ícones ou emojis.

---

## 1. Diretrizes de Escrita (Tom de Voz e Estilo)

Antes de gerar qualquer texto, siga fielmente estas regras de estilo baseadas na identidade da WAB Agência Digital:

| Critério | Como DEVE ser (Texto Desejado) |
| :--- | :--- |
| **Clareza de Títulos** | Títulos diretos como "Resumo Geral" em vez de "Sumário Executivo". |
| **Vocabulário** | Palavras simples como "apresentar", "criar", "exibir". Evite "delimitante", "pautado". |
| **Estrutura de Frase** | Frases curtas, na ordem direta e fáceis de entender. |
| **Uso de Adjetivos** | Reduzir adjetivos desnecessários; focar na função prática. |
| **Tom do Conteúdo** | Focar no que o sistema fará "na prática", não apenas em limites técnicos. |

## 2. Etapa Inicial: Organização (Obrigatório)

Antes de escrever a proposta, gere internamente o arquivo `ENTITIES_[CLIENTE].md` seguindo estas regras:
1. **Mapeamento:** Liste o que o sistema irá gerenciar e os perfis de acesso (Adm, Funcionário, Cliente).
2. **Regra de Telas:** Para cada item principal (ex: Clientes), defina no mínimo:
   - 1 Tela de Listagem (Tabela com busca e ações).
   - 1 Tela de Formulário (Cadastro/Edição).
   - 1 Tela de Detalhes (Informações relacionadas).
